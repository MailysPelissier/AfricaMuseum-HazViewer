Installation et configuration de Geoserver :
Installer le fichier d'installation Geoserver : https://geoserver.org/
Avoir un environnement Java 11 ou 17 : https://adoptium.net/
Mettre le chemin de cet environnement lors du setup ex : C:\Program Files\Eclipse Adoptium\jre-11.0.26.4-hotspot
Il faut également autoriser l’accès à votre serveur (Jetty) depuis un client avec les CORS :
https://docs.geoserver.org/latest/en/user/production/container.html#enable-cors-for-jetty-binary-installer


Paramètres de Geoserver (par défaut) :
Username : admin
Password : geoserver
Port : 8080


Création d'un Workspace (ici nommé webGIS)
Créer d’abord un nouveau Workspace (webGIS)
Ce Workspace contiendra 3 stores, ici nommés PostGIS, rivers et landslide susceptibility, et 5 couches (events, paragraphs, citizen_observer, HydroRIVERS_flow5max, landslide_susceptibility)


Création du store PostGIS
Ce store contiendra 3 couches (events, paragraphs, citizen_observer)
Les couches peuvent être créées depuis les tables pgadmin ou depuis les fichiers .sql
events.sql : "C:\Mailys\data\hazminer\events.sql"
paragraphs.sql : "C:\Mailys\data\hazminer\paragraphs.sql"
citizen_observer.sql : "C:\Mailys\data\citizen_observer\citizen_observer.sql"
Créer un Store, lié à votre base PostGIS (PostGIS)
Créer les layers (la géométrie est utilisée automatiquement) (events, paragraphs, citizen_observer)
Layer : 
Declared SRS: EPSG:4326
Bounding Boxes : cliquer sur Compute from SRS bounds puis sur Compute from native bounds
Vérifier que la clé primaire n'apparait pas dans les propriétés
Vérifier que ça fonctionne avec le layer preview


Création du store rivers
Ce store contiendra une couche (HydroRIVERS_flow5max)
Créer un Store format shapefile (rivers)
Créer la layer (HydroRIVERS_flow5max)
Connection parameters : url du .shp en local (par exemple: "C:\Mailys\data\rivers\HydroRIVERS_v10_shp\HydroRIVERS_flow5max.shp")
Appuyer sur publier la couche


Création du store landslide susceptibility
Ce store contiendra une couche (landslide susceptibility)
Créer un Store format GeoTIFF (landslide susceptibility)
Créer la layer (landslide susceptibility)
Connection parameters : url du .tif en local (par exemple: "C:\Mailys\data\landslide_susceptibility\landslide_susceptibility.tif")
Appuyer sur publier la couche

Créer un style (landslide_style)
Format SLD
Editeur SLD:
<StyledLayerDescriptor version="1.0.0"
xsi:schemaLocation="http://www.opengis.net/sld StyledLayerDescriptor.xsd"
xmlns="http://www.opengis.net/sld"
xmlns:ogc="http://www.opengis.net/ogc"
xmlns:xlink="http://www.w3.org/1999/xlink"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <NamedLayer>
    <Name>landslide_style</Name>
    <UserStyle>
      <Title>Landslide Susceptibility</Title>
      <FeatureTypeStyle>
        <Rule>
          <RasterSymbolizer>
            <Opacity>1.0</Opacity>
            <ColorMap>
              <ColorMapEntry color="#FFFFFF" quantity="0" label="0" opacity="0"/>
              <ColorMapEntry color="#FFF8DC" quantity="1" label="1" opacity="1"/>
              <ColorMapEntry color="#DEB887" quantity="2" label="2" opacity="1"/>
              <ColorMapEntry color="#CD853F" quantity="3" label="3" opacity="1"/>
              <ColorMapEntry color="#A0522D" quantity="4" label="4" opacity="1"/>
              <ColorMapEntry color="#8B4513" quantity="5" label="5" opacity="1"/>
            </ColorMap>
          </RasterSymbolizer>
        </Rule>
      </FeatureTypeStyle>
    </UserStyle>
  </NamedLayer>
</StyledLayerDescriptor>
L'appliquer à la couche : dans l'onglet publishing de la couche, changer le default style
